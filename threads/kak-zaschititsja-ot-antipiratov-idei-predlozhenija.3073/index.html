Все доброго времени суток, в очередной раз антипираты прислали письма на мои самые посещаемые страницы .
Скрыл страницы, поставил 404 для надзорных органов во WHOIS, добавил все их ip в раздел "Заблокировать доступ по IP".


Прочёл статью про Group-IB

В следствии чего родилась идея уникализации целевых url:
  1. Вариант - Что если, страницу с плеером для каждого пользователя открывать по уникальному адресу? токену сформированному к примеру по user-agent, в таком случае когда будет приходить письмо счастья, CMS автоматом будет закрывать доступ к странице с враждебным токеном, а все нормальные люди и дальше будут наслаждаться просмотром.

  2. Вариант - Возможно токен из user-agent использовать в качестве поддомена, тогда можно будет убить весь поддомен с вредным токеном, тем самым закрыв доступ конкретному человеку (боту) закрывать доступ на весь сайт, ибо нечего им тут шастать
    Только поддоменов вроде можно создавать определеннов количество, в этом проблема.

Как идея? Сильно не бейте, это только теория...
Правда есть один минус если это вскроется то в блок сайт точно попадет

по первому пункту есть пара возможных минусов.
ПФ для поисковиков будут плохие - многие пользователи будут уходить сразу с сайта (из-за редиректа или просто закрывать вкладку с описанием фильма, оставляя вкладку с плеером)
